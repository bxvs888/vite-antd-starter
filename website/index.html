快速开始

使用 Vite+ 的统一 CLI，一条命令创建项目，运行即可。

Getting Started

Create a project with one command using the Vite+ unified CLI, then run it immediately.

  1. 1
    安装 Vite+全局安装统一工具链。
    Install Vite+Install the unified toolchain globally.
    npm install -g vite-plus
  2. 2
    交互式创建运行创建命令，选择想要的模板。
    Create InteractivelyRun the create command and pick your template.
    vp create @brandonxiang
  3. 3
    或指定模板跳过选择，直接指定模板名称。
    Or Target a TemplateSkip the picker and scaffold a specific template directly.
    vp create @brandonxiang:viteplus-antd my-admin
vp create @brandonxiang:viteplus-antd-ssr my-ssr-app
vp create @brandonxiang:viteplus-antd-mobile my-mobile
vp create @brandonxiang:viteplus-fastify my-api
  4. 4
    启动开发服务进入项目目录，安装依赖，开始开发。
    Start DevelopingEnter the project directory, install, and launch the dev server.
    cd my-admin
vp install
vp dev

常见问题

关于模板目录的快速解答。

FAQ

Quick answers about the template catalog.

这些模板适合生产环境吗？
是的。每个模板都经过 vp check 和 vp test 验证，Lighthouse 达到满分基线，TypeScript 严格模式，适合直接用于生产项目。
Are these templates production-ready?
Yes. Every template passes vp check and vp test, achieves a Lighthouse 100 baseline, uses strict TypeScript, and is designed for production use from day one.
Vite+ 和 Vite 有什么区别？
Vite+ 是在 Vite 基础上构建的统一工具链，封装了 Rolldown、Vitest、Oxlint、Oxfmt，提供 vp dev、vp build、vp check、vp test 等统一命令。
What's the difference between Vite+ and Vite?
Vite+ is a unified toolchain built on Vite, wrapping Rolldown, Vitest, Oxlint, and Oxfmt into a single CLI. Commands like vp dev, vp build, vp check, and vp test replace multiple tools with one interface.
可以同时使用多个模板吗？
当然。模板彼此独立，你可以在一个项目中混合使用 Antd 后台和 Fastify 服务端。它们共用同一个 Vite+ 工具链，开发者体验一致。
Can I use multiple templates together?
Absolutely. Templates are independent. You could pair an Antd SPA frontend with a Fastify backend — they share the same Vite+ toolchain for a consistent developer experience.
如何提交 Issue 或贡献？
请访问 GitHub 仓库 提交 Issue 或 Pull Request。我们欢迎社区贡献。
How do I submit an issue or contribute?
Visit the GitHub repository to open issues or submit pull requests. Community contributions are welcome.
